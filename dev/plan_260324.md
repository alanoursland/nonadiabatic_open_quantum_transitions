# Implementation Plan: Nonadiabatic Line-Shape Comparison Against HITRAN CO₂ Data

## Current State

The theoretical framework is complete. The nonadiabatic relaxation matrix is $\mathbf{W}^{\text{nad}} = \mathbf{W}^{\text{standard}} \mathcal{F}$, where $\mathcal{F} = (\mathbf{I} - \mathbf{M}) \otimes (\mathbf{I}^* - \mathbf{M}^*)$ is a filter superoperator built from the mean collisional field. The sum rule is verified, detailed balance is understood, and the correction structure (first-order inter-branch, second-order intra-branch) is derived.

The codebase has a `RotationalSystem` class, a `CollisionalLorentzian` spectral density, and the full Redfield/CGME/secular dissipator infrastructure (265 tests passing). What's missing is the machinery to construct $\mathbf{M}$, build $\mathcal{F}$, assemble $\mathbf{W}^{\text{nad}}$, compute the line shape, and compare against HITRAN data.

This plan specifies the implementation in five work packages ordered by dependency.

## Work Package 1: Mean-Field Mixing Matrix $\mathbf{M}$

**New file:** `src/mixing_matrix.py`

### What it does

Constructs the adiabatic mixing matrix $\mathbf{M}$ for a rigid rotor under a mean collisional field. The matrix elements are:

$$M_{kj} = \frac{\langle k | \langle V_{\text{int}} \rangle | j \rangle}{E_j - E_k} \quad (k \neq j), \qquad M_{kk} = 0$$

For a $P_2(\cos\theta)$-dominated intermolecular potential (the leading anisotropy for linear molecule–atom collisions), $\langle V_{\text{int}} \rangle$ connects states with $\Delta J = \pm 2$. The coupling strength is $V = \gamma_L P$ where $\gamma_L$ is the pressure-broadening coefficient and $P$ is the buffer gas pressure.

### Implementation

```python
class MixingMatrix:
    def __init__(self, rotational_system, pressure_atm, gamma_L_cm1_atm,
                 anisotropy_order=2):
        """
        rotational_system: RotationalSystem instance (provides energy levels)
        pressure_atm: buffer gas pressure in atm
        gamma_L_cm1_atm: pressure-broadening coefficient in cm⁻¹/atm
        anisotropy_order: dominant Legendre order of the potential (default 2)
        """

    def get_M(self):
        """Return the mixing matrix M (dim × dim) in the rotational eigenbasis."""

    def get_perturbation_ratios(self):
        """Return V/ΔE for each J, for diagnostics."""
```

The matrix elements involve angular momentum coupling coefficients (Clebsch-Gordan or 3-j symbols) for the $\langle J | P_2 | J' \rangle$ selection rule. For $P_2$: $\Delta J = 0, \pm 2$, but $\Delta J = 0$ gives diagonal elements (set to zero in $\mathbf{M}$). The $\Delta J = \pm 2$ matrix elements are:

$$\langle J+2 | P_2(\cos\theta) | J \rangle = \frac{[(J+1)(J+2)]^{1/2} [(2J+1)(2J+5)]^{1/2}}{(2J+3)} \times (\text{3-j symbol})$$

Use `scipy.special` for the 3-j symbols, or hardcode the analytical formula for $P_2$ coupling which is well-known.

### Tests

- $\mathbf{M}$ is zero on the diagonal
- $\mathbf{M}$ is nonzero only for $|k - j| = 2$ (for $P_2$ anisotropy)
- $M_{kj}$ scales as $1/(E_j - E_k) \propto 1/(J+1)$ for adjacent states
- $\mathbf{M}$ scales linearly with pressure
- $\mathbf{M} \to 0$ as $P \to 0$
- Perturbation ratios match the Phase 0A estimates (e.g., $V/\Delta E = 0.449$ for CO₂ at 5 atm, $J = 0$)
- CO₂ symmetry: $M_{kj}$ is zero for $\Delta J = \pm 1$ (only even $\Delta J$ for homonuclear-like symmetry)

### Estimated effort

1–2 days.

## Work Package 2: Nonadiabatic Filter Superoperator $\mathcal{F}$

**New file:** `src/nonadiabatic_filter.py`

### What it does

Constructs the filter superoperator $\mathcal{F} = (\mathbf{I} - \mathbf{M}) \otimes (\mathbf{I}^* - \mathbf{M}^*)$ that operates in line space (Liouville space). Computes the nonadiabatic relaxation matrix $\mathbf{W}^{\text{nad}} = \mathbf{W}^{\text{standard}} \mathcal{F}$.

### Key design decision: line-space representation

The filter $\mathcal{F}$ operates in the space of optical coherences. Each optical coherence is labeled by its upper and lower states: $|J'\rangle\langle J|$ where $J' = J+1$ (R-branch) or $J' = J-1$ (P-branch). The line-space dimension is the number of included transitions.

$\mathcal{F}$ maps line-space vectors to line-space vectors. Its matrix elements in line space are:

$$\mathcal{F}_{(J'_a, J_a), (J'_b, J_b)} = [(\mathbf{I} - \mathbf{M})]_{J'_a, J'_b} \times [(\mathbf{I}^* - \mathbf{M}^*)]_{J_a, J_b}$$

where $(J'_a, J_a)$ labels the row (output line) and $(J'_b, J_b)$ labels the column (input line).

For the standard intra-branch case ($J'_a = J_a + 1$ and $J'_b = J_b + 1$, both R-branch), the diagonal of $\mathcal{F}$ is 1 and the off-diagonal elements come from $\mathbf{M}$.

For inter-branch coupling (e.g., $J'_a = J_a + 1$ is R-branch and $J'_b = J_b - 1$ is P-branch), the elements come from the cross-terms in $\mathbf{M}$.

### Implementation

```python
class NonadiabaticFilter:
    def __init__(self, mixing_matrix, line_indices):
        """
        mixing_matrix: MixingMatrix instance
        line_indices: list of (upper_J, lower_J) tuples defining the line space
                      e.g., [(1,0), (2,1), (3,2), ...] for R-branch
                      or [(1,0), (2,1), ..., (0,1), (1,2), ...] for R+P
        """

    def get_F_matrix(self):
        """Return the filter as a matrix in line space (n_lines × n_lines)."""

    def apply(self, W_standard):
        """Compute W_nad = W_standard @ F. Returns the nonadiabatic W-matrix."""

    def get_inter_branch_block(self):
        """Extract the R-P coupling block of F for inspection."""
```

### Tests

- $\mathcal{F} \to \mathbf{I}$ as $P \to 0$ ($\mathbf{M} \to 0$)
- $\mathcal{F}$ is not idempotent ($\mathcal{F}^2 \neq \mathcal{F}$), confirming it's a filter not a projector
- The diagonal elements of $\mathcal{F}$ in the R-branch block are $\approx 1 - O(|M|^2)$
- The R-P block of $\mathcal{F}$ is nonzero (inter-branch coupling exists)
- The R-P block elements scale linearly with pressure (first order in $\mathbf{M}$)
- The R-R off-diagonal elements scale quadratically with pressure (second order)
- Sum rule: $\mathbf{v}^T \mathbf{W}^{\text{standard}} \mathcal{F} = 0$ for the population-weighted dipole vector $\mathbf{v}$

### Estimated effort

1–2 days.

## Work Package 3: Line-Shape Calculator

**New file:** `src/line_shape.py`

### What it does

Computes the pressure-broadened absorption line shape from the resolvent formula:

$$F(\nu) = \frac{1}{\pi} \operatorname{Im}\left[\mathbf{d}^T \left( (\nu - \boldsymbol{\nu}_0) \mathbf{I} - i\mathbf{W} \right)^{-1} \boldsymbol{\rho}_{\text{eq}} \right]$$

and the Rosenkranz first-order line-mixing $Y$-parameters:

$$Y_J = \frac{2}{P \gamma_J} \sum_{J' \neq J} \frac{W_{JJ'} \, d_{J'} \, \rho_{J'}}{d_J \, \rho_J} \cdot \frac{1}{\nu_J - \nu_{J'}}$$

### Implementation

```python
class LineShapeCalculator:
    def __init__(self, line_positions_cm1, line_strengths, W_matrix,
                 temperature_K=296.0):
        """
        line_positions_cm1: array of line center frequencies (cm⁻¹)
        line_strengths: array of d_J * rho_J products
        W_matrix: relaxation matrix (n_lines × n_lines, complex, cm⁻¹)
        temperature_K: temperature for thermal populations
        """

    def absorption(self, nu_grid_cm1):
        """Compute F(nu) on the given frequency grid. Returns array."""

    def Y_parameters(self):
        """Compute Rosenkranz Y_J for each line. Returns array."""

    def lorentzian_no_mixing(self, nu_grid_cm1):
        """Compute the no-line-mixing (isolated lines) spectrum for comparison."""
```

The resolvent formula requires inverting $(\nu \mathbf{I} - \boldsymbol{\nu}_0 - i\mathbf{W})$ at each frequency point. For a $40 \times 40$ matrix, this is $\sim 1 \mu$s per frequency point — trivial even for a dense grid.

### Tests

- For $\mathbf{W} = 0$ (no collisions), $F(\nu)$ reduces to a sum of delta functions at line positions
- For diagonal $\mathbf{W}$ (no line mixing), $F(\nu)$ reduces to a sum of Lorentzian profiles
- The integrated area $\int F(\nu) d\nu$ is independent of $\mathbf{W}$ (sum rule consequence)
- $Y_J = 0$ when $\mathbf{W}$ is diagonal (no line mixing)
- For a known 2-line system with analytical line-mixing solution, the numerical result matches
- $F(\nu) \geq 0$ for all $\nu$ when $\mathbf{W}$ is a physical relaxation matrix

### Estimated effort

1–2 days.

## Work Package 4: HITRAN Data Interface

**New file:** `src/hitran_data.py`

### What it does

Downloads and parses CO₂ line parameters from the HITRAN database. Provides the molecular data needed for the comparison: line positions, intensities, broadening coefficients, pressure shifts, and $Y$-parameters.

### Implementation

```python
class HITRANData:
    def __init__(self, molecule='CO2', isotopologue=1, band='nu3',
                 branch='R', J_max=20):
        """
        Loads CO₂ ν₃ band parameters from HITRAN.
        Uses HAPI (HITRAN Application Programming Interface) if available,
        otherwise reads from a local data file.
        """

    def line_positions(self):
        """Return line center frequencies in cm⁻¹."""

    def line_intensities(self, T=296.0):
        """Return line intensities at temperature T."""

    def broadening_coefficients(self, perturber='N2'):
        """Return γ_J (half-width) and δ_J (shift) in cm⁻¹/atm."""

    def Y_parameters(self):
        """Return first-order line-mixing Y-parameters if available."""

    def transition_dipoles(self):
        """Return |d_J| for each transition."""

    def thermal_populations(self, T=296.0):
        """Return (2J+1) exp(-E_J/kT) / Z for each lower state."""
```

### Data source options

1. **HAPI (preferred):** The HITRAN Application Programming Interface is a Python package that downloads data directly from hitran.org. Install with `pip install hitran-api`. This gives programmatic access to all HITRAN parameters.

2. **Local data file (fallback):** If network access is unavailable, hardcode the CO₂ $\nu_3$ R-branch and P-branch parameters for $J = 0$–$20$ from the HITRAN2020 tables. The data is small (~40 lines × ~10 parameters = ~400 numbers).

3. **Hartmann's $W$-matrix:** This is not in HITRAN. It must be extracted from Hartmann's publications (monograph tables or JQSRT supplementary data). Hardcode the published values or read from a data file.

### Tests

- Line positions match published HITRAN values to better than 0.001 cm⁻¹
- Line intensities are positive and decrease with increasing $J$ (for $J$ beyond the Boltzmann peak)
- Broadening coefficients are positive and of order 0.05–0.10 cm⁻¹/atm
- Thermal populations sum to 1 (within the included $J$ range)
- The number of R-branch and P-branch lines matches expectations for the band

### Estimated effort

2–3 days (including data extraction from literature for the $W$-matrix).

## Work Package 5: Comparison Script

**New file:** `src/hitran_comparison.py`

### What it does

Puts it all together. For each pressure in [1, 2, 3, 5, 7, 10] atm:

1. Builds $\mathbf{M}$ from `MixingMatrix`
2. Builds $\mathcal{F}$ from `NonadiabaticFilter`
3. Loads $\mathbf{W}^{\text{standard}}$ from `HITRANData` (Hartmann's values)
4. Computes $\mathbf{W}^{\text{nad}} = \mathbf{W}^{\text{standard}} \mathcal{F}$
5. Computes $Y_J^{\text{standard}}$ and $Y_J^{\text{nad}}$ from `LineShapeCalculator`
6. Computes absorption line shapes $F^{\text{standard}}(\nu)$ and $F^{\text{nad}}(\nu)$
7. Compares against HITRAN $Y$-parameters and (if available) measured absorption spectra
8. Produces diagnostic plots and $\chi^2$ statistics

### Output

**Table 1:** For each $R(J)$ and $P(J)$ at each pressure: $Y_J^{\text{HITRAN}}$, $Y_J^{\text{standard}}$, $Y_J^{\text{nad}}$, residuals.

**Table 2:** Inter-branch coupling matrix elements $[W^{\text{nad}}]_{R(J), P(J')}$ at each pressure.

**Figure 1:** $Y_J$ vs $J$ at 5 atm: HITRAN data, standard prediction, nonadiabatic prediction.

**Figure 2:** $Y_J^{\text{HITRAN}} - Y_J^{\text{standard}}$ residual vs $J$, overlaid with the predicted nonadiabatic correction $Y_J^{\text{standard}} - Y_J^{\text{nad}}$. If these curves match, the nonadiabatic framework explains the residuals.

**Figure 3:** Absorption line shape near the band center at 5 atm, showing the standard and nonadiabatic predictions against measured data.

**Figure 4:** Pressure scaling of the correction: $|Y_J^{\text{standard}} - Y_J^{\text{nad}}|$ vs $P$ for $J = 0, 2, 4$, checking the $P^2$ scaling prediction.

**Figure 5:** Inter-branch coupling magnitude vs $P$, checking the $P^1$ scaling prediction.

### Phase 1 gate (before full comparison)

Before running the full comparison, compute $\mathbf{M}$ and $\mathcal{F}$ at 5 atm and inspect the matrix elements. Verify:

- The inter-branch block of $\mathcal{F}$ has elements of order 0.1 at low $J$ (as predicted by Phase 0A)
- The intra-branch correction to $\mathcal{F}$ is of order 0.04 at $J = 0$ (the $|M|^2$ correction)
- The sum rule is satisfied numerically

If these magnitudes are as expected, proceed with the full comparison. If they're orders of magnitude smaller (because the angular momentum algebra suppresses the coupling), reassess before investing in data extraction.

### Tests

- At $P = 0$: $Y_J^{\text{nad}} = Y_J^{\text{standard}}$ exactly
- At large $J$ ($J > 15$): $Y_J^{\text{nad}} \approx Y_J^{\text{standard}}$ to within 0.1%
- The correction $|Y^{\text{standard}} - Y^{\text{nad}}|$ increases with pressure
- The integrated line shape area is the same for standard and nonadiabatic predictions (sum rule)
- $F(\nu) \geq 0$ for all $\nu$ for both predictions

### Estimated effort

3–5 days.

## Phase 1 Gate

After WP1 and WP2 are complete but before investing in WP4 (data extraction) and WP5 (full comparison):

Compute $\mathbf{M}$ and $\mathcal{F}$ for CO₂ at 5 atm. Inspect:

1. **Inter-branch coupling magnitude.** If the R-P block of $\mathcal{F}$ has elements $\sim 0.1$ at low $J$, the first-order prediction is alive. If $\sim 0.001$, the angular momentum algebra has suppressed it and only the second-order intra-branch correction is testable.

2. **Intra-branch correction magnitude.** The R-R off-diagonal correction should be $\sim (V/\Delta E)^2 \sim 0.04$ at $J = 0$. If it's much smaller, the Phase 0A estimate was wrong and the whole comparison may not be feasible.

3. **Sum rule.** Verify numerically with a toy $\mathbf{W}^{\text{standard}}$.

This gate takes one day after WP2 is done. It determines whether to proceed with the full data comparison or pivot.
