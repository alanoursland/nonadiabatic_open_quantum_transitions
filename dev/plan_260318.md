# Codebase Refactoring Plan: From Transmon Skeleton to NMR $R_{1\rho}$ Simulator

## Current State

The codebase has five source files and four test files. The architecture is sound: `System` defines the Hamiltonian, `PlateauPulse` defines the drive (including analytical derivatives), `ThermalBath` defines the dissipation, `Simulator` integrates the master equation via RK4, and `Result` extracts observables. Tests check trace preservation, Hermiticity, population decay direction, and commutator algebra.

The problems are: (1) the `nonadiabatic_cgme` method is a placeholder that produces identical output to `lindblad`, (2) the bath is phenomenological ($T_1$/$T_2$ inputs) rather than microscopic (spectral density), so both methods use the same hardcoded rates, (3) the system is a transmon qutrit, not an NMR spin system, and (4) no nonadiabatic decomposition is implemented anywhere.

The goal is to turn this into a simulator that can compute $R_{1\rho}$ for an NMR spin system using three methods — standard Redfield, secular Lindblad, and nonadiabatic CGME — and compare the results against published experimental data.

---

- **`Result` class** (`simulator.py` lines 4–21). The population extraction and plotting logic is fine. Will need minor additions (coherence extraction, $R_{1\rho}$ fitting) but the core is reusable.

## Work Package 1: New `SpinSystem` Class

**Replaces:** `system.py`

**File:** `spin_system.py`

The current `System` class defines a generic multi-level system with energy levels and a dipole matrix. For NMR, the system is a collection of coupled spin-1/2 particles in a static magnetic field, and the Hamiltonian has specific structure.

**Implementation:**

```
class SpinSystem:
    def __init__(self, spins, B0_field, J_couplings, csa_tensors, device='cpu'):
        """
        spins: list of dicts, each with 'nucleus' ('15N', '1H', etc.),
               'chemical_shift_ppm', 'gamma' (gyromagnetic ratio)
        B0_field: static field strength in Tesla
        J_couplings: dict mapping (i,j) pairs to scalar coupling in Hz
        csa_tensors: dict mapping spin index to CSA in ppm
        """
```

**Must compute:**

- Larmor frequencies $\omega_i = -\gamma_i B_0 (1 + \delta_i \times 10^{-6})$ for each spin, where $\delta_i$ is the chemical shift in ppm.
- The Zeeman Hamiltonian $H_0 = \sum_i \omega_i I_{z,i}$ in the product basis.
- The scalar coupling Hamiltonian $H_J = \sum_{i<j} 2\pi J_{ij} \mathbf{I}_i \cdot \mathbf{I}_j$.
- The full unperturbed Hamiltonian $H_0 + H_J$.
- Bohr frequencies from the eigenvalues of $H_0 + H_J$.
- Spin operator matrices ($I_x$, $I_y$, $I_z$ for each spin) in the product basis, needed for constructing the drive and the system-bath coupling.

For a two-spin $^{15}$N-$^1$H system, the Hilbert space is $2^2 = 4$ dimensional. The product basis states are $|\alpha\alpha\rangle$, $|\alpha\beta\rangle$, $|\beta\alpha\rangle$, $|\beta\beta\rangle$. All operators are 4×4 matrices.

**Deliverables:**
- `spin_system.py` with `SpinSystem` class
- `test_spin_system.py` verifying: correct Larmor frequencies, Hermiticity of $H_0$, correct Bohr frequencies for a known two-spin system, spin operator commutation relations $[I_x, I_y] = iI_z$

**Estimated effort:** 1–2 days.

## Work Package 2: `SpinLockDrive` Class

**Replaces:** the transmon-specific use of `PlateauPulse`

**File:** `signals.py` (extend, don't replace)

The spin-lock in NMR is an RF field applied transverse to $B_0$ after a 90° preparation pulse. In the rotating frame at the carrier frequency, it appears as a constant field along $x$ (or $y$). The perturbation Hamiltonian is:

$$V(t) = \omega_1 \cdot f(t) \cdot I_{x,N}$$

where $\omega_1 = \gamma_N B_1$ is the spin-lock power in rad/s, $f(t)$ is the envelope (ramp up, plateau, ramp down — exactly the `PlateauPulse` envelope), and $I_{x,N}$ is the $x$-operator of the $^{15}$N spin.

**Implementation:**

```
class SpinLockDrive:
    def __init__(self, spin_lock_power_hz, target_spin_index, 
                 offset_hz=0.0, ramp_time_us=0.0, plateau_time_us=50.0):
        """
        spin_lock_power_hz: omega_1 / (2*pi) in Hz
        target_spin_index: which spin the lock acts on
        offset_hz: off-resonance offset in Hz
        ramp_time_us: ramp duration in microseconds
        plateau_time_us: plateau duration in microseconds
        """
```

This class wraps the existing `PlateauPulse` envelope math but parameterizes it in NMR units (Hz, microseconds) and knows which spin operator it couples to.

The `derivative()` method returns $dV/dt$, which is nonzero only during the ramps (the envelope derivative times $I_{x,N}$). During the plateau, $dV/dt = 0$ exactly — this is the key property that makes the nonadiabatic populations constant.

**Key detail:** For off-resonance spin-lock, $V(t)$ includes both the transverse component $\omega_1 I_x$ and the offset component $\Delta\omega I_z$ in the rotating frame. The effective field is tilted, and the nonadiabatic decomposition operates relative to this tilted effective field.

**Deliverables:**
- Extended `signals.py` with `SpinLockDrive`
- `test_spin_lock.py` verifying: correct perturbation matrix, zero derivative during plateau, correct off-resonance effective field angle

**Estimated effort:** 1 day.

## Work Package 3: Microscopic `SpectralDensityBath` Class

**Replaces:** `bath.py`

**File:** `bath.py` (rewrite)

This is the most critical change. The current `ThermalBath` takes measured $T_1$/$T_2$ as inputs and builds Lindblad operators directly. This makes both methods use the same rates, defeating the purpose of the comparison. The new bath must compute rates from first principles (spectral density + temperature), so the Dirac and nonadiabatic frameworks can derive *different* rates from the same underlying bath.

**Implementation:**

```
class SpectralDensityBath:
    def __init__(self, spectral_density_fn, temperature_K, 
                 coupling_operators, device='cpu'):
        """
        spectral_density_fn: callable J(omega) -> float
            The bath spectral density. For Lipari-Szabo model-free:
            J(w) = (2/5) * [S^2 * tau_c/(1+w^2*tau_c^2) 
                    + (1-S^2)*tau_e/(1+w^2*tau_e^2)]
        temperature_K: bath temperature in Kelvin
        coupling_operators: list of system operators S_alpha that 
            couple to the bath (e.g., dipolar, CSA)
        """
```

**Must compute:**

1. **Bath correlation function one-sided Fourier transforms:**
   $$\Gamma(\omega) = \int_0^\infty e^{i\omega\tau} C(\tau) d\tau$$
   For a spectral density $J(\omega)$, this gives $\Gamma(\omega) = J(\omega)[n(\omega) + 1] + iS(\omega)$ where $n(\omega)$ is the Bose-Einstein distribution and $S(\omega)$ is the Lamb shift integral.

2. **Full Redfield tensor $R_{abcd}$** from the bath correlation functions evaluated at the system's Bohr frequencies, using the matrix elements of the coupling operators in the energy eigenbasis.

3. **Coarse-grained Redfield tensor $R_{abcd}^{\text{CG}}$** by applying the sinc damping:
   $$R_{abcd}^{\text{CG}} = R_{abcd} \cdot \text{sinc}\!\left(\frac{(\omega_{ab} - \omega_{cd})\Delta\tau}{2}\right)$$

4. **Secular Redfield tensor** (the $\Delta\tau \to \infty$ limit of coarse-graining, which zeroes all non-secular terms).

5. **Three dissipator methods:**
   - `get_redfield_dissipator(system, rho)` — full non-secular Redfield
   - `get_secular_lindblad_dissipator(system, rho)` — secular approximation
   - `get_cgme_dissipator(system, rho, delta_tau)` — coarse-grained, positivity-preserving

Each dissipator acts on the density matrix $\rho$ and returns $\mathcal{D}[\rho]$.

**For NMR specifically:** The coupling operators for a $^{15}$N-$^1$H pair are the dipolar Hamiltonian (rank-2 spherical tensor components modulated by molecular tumbling) and the $^{15}$N CSA Hamiltonian. The spectral density is the Lipari-Szabo model-free form parameterized by the overall correlation time $\tau_c$, the internal correlation time $\tau_e$, and the order parameter $S^2$.

**Deliverables:**
- Rewritten `bath.py` with `SpectralDensityBath`
- `spectral_densities.py` with common spectral density models (`LipariSzabo`, `SimpleLorentzian`, `OhmicDrude`)
- `test_bath.py` rewritten to verify: detailed balance ($\gamma_{\downarrow}/\gamma_{\uparrow} = e^{\hbar\omega/k_BT}$), correct high-temperature limit, trace preservation of all three dissipators, Hermiticity preservation, and agreement of the secular Lindblad dissipator with the old phenomenological `ThermalBath` for a known system

**Estimated effort:** 3–5 days. This is the hardest module.

## Work Package 4: `NonadiabaticDecomposition` Class

**New file:** `nonadiabatic.py`

This is the physics that doesn't exist in the current codebase. It implements the Landau-Lifshitz integration by parts to compute the nonadiabatic coefficients $b_k(t)$ from the history of the perturbation.

**Implementation:**

```
class NonadiabaticDecomposition:
    def __init__(self, system, device='cpu'):
        """
        Tracks the adiabatic and nonadiabatic coefficients over time.
        Must be called sequentially as the simulation progresses.
        """
        self.system = system
        self.dim = system.dim
        # Running integral for b_k(t)
        self.b_coeffs = torch.zeros(self.dim, dtype=torch.complex128)
        self.b_coeffs[0] = 1.0  # start in ground state
```

**Must compute at each time step:**

1. **Adiabatic coefficients** $a_k(t)$ from the instantaneous perturbation:
   $$a_k^{(1)}(t) = \frac{\langle k | V(t) | 0 \rangle}{E_k - E_0} \, e^{i\omega_{k0} t}$$

2. **Nonadiabatic coefficients** $b_k(t)$ by accumulating the integral:
   $$b_k(t) = -\frac{1}{i\hbar \cdot i\omega_{k0}} \int_{-\infty}^{t} \frac{d}{dt'}\langle k | V(t') | 0 \rangle \, e^{i\omega_{k0} t'} \, dt'$$
   
   In practice, this integral is accumulated numerically using the trapezoidal rule at each time step. The integrand at time $t'$ is $\frac{d}{dt'}\langle k|V(t')|0\rangle \cdot e^{i\omega_{k0}t'}$, which requires the matrix elements of $dV/dt'$ — available from the `SpinLockDrive.derivative()` method.

3. **Nonadiabatic density matrix** $\sigma_{\text{nad}}$:
   $$[\sigma_{\text{nad}}]_{kl} = b_k(t) b_l^*(t)$$

**Key methods:**
- `step(t, dt, V_matrix, dV_dt_matrix)` — advance the integral by one time step
- `get_adiabatic_coefficients(t, V_matrix)` — return $a_k(t)$
- `get_nonadiabatic_coefficients()` — return current $b_k(t)$
- `get_nonadiabatic_density_matrix()` — return $\sigma_{\text{nad}}(t)$
- `get_nonadiabatic_populations()` — return $|b_k(t)|^2$

**Deliverables:**
- `nonadiabatic.py` with `NonadiabaticDecomposition`
- `test_nonadiabatic.py` verifying:
  - During a plateau ($dV/dt = 0$), calling `step()` does not change $b_k$
  - For a known analytically solvable case (two-level system with sudden turn-on), $b_k(t)$ matches the analytical result
  - $|a_k(t)|^2 + |b_k(t)|^2$ is consistent with $|c_k(t)|^2$ (the decomposition is complete)
  - Gauge invariance: compute $b_k(t)$ for the same physical scenario in two different representations and verify they agree

**Estimated effort:** 2–3 days.

## Work Package 5: Refactored `Simulator`

**Replaces:** `simulator.py`

The current simulator has a single propagation loop that calls `liouvillian()` at each step. The nonadiabatic method requires additional state (the accumulated integral for $b_k$) that the current loop doesn't track.

**Three methods, three different physics:**

**Method `redfield`:** Standard non-secular Redfield.
- Coherent evolution: $-i[H_0 + V(t), \rho]$
- Dissipation: Full Redfield dissipator from `SpectralDensityBath.get_redfield_dissipator()`
- This is the Dirac-basis, non-secular, may-violate-positivity method.

**Method `secular_lindblad`:** Standard secular Lindblad.
- Coherent evolution: $-i[H_0 + V(t), \rho]$
- Dissipation: Secular Lindblad dissipator from `SpectralDensityBath.get_secular_lindblad_dissipator()`
- This is the Dirac-basis, secular, guaranteed-positive method.

**Method `nonadiabatic_cgme`:** The new method.
- At each time step, call `NonadiabaticDecomposition.step()` to update $b_k(t)$.
- Coherent evolution: $-i[H_0 + V_{\text{nad}}(t), \sigma_{\text{nad}}]$ where $V_{\text{nad}}$ is the nonadiabatic part of the driving (excluding the adiabatic polarization).
- Dissipation: Coarse-grained dissipator from `SpectralDensityBath.get_cgme_dissipator()` acting on $\sigma_{\text{nad}}$.
- The propagated object is $\sigma_{\text{nad}}$, not $\rho$.

**Important structural change:** The `run()` method for `nonadiabatic_cgme` must instantiate a `NonadiabaticDecomposition` object before the loop and call its `step()` method inside the loop. This is stateful — the integrator carries forward the accumulated $b_k$ values. The other two methods remain stateless (the Liouvillian depends only on the current $\rho$ and $t$).

**Remove** the trace renormalization `rho = rho / torch.trace(rho)`. Replace it with an assertion that checks trace drift and raises an error if it exceeds a tolerance (e.g., $|1 - \text{Tr}(\rho)| > 10^{-6}$). This ensures bugs in the dissipator are caught, not hidden.

**Deliverables:**
- Rewritten `simulator.py`
- `test_simulator.py` rewritten to verify:
  - All three methods preserve trace (to machine precision, not by renormalization)
  - All three methods preserve Hermiticity
  - For zero drive ($V = 0$), all three methods give identical results (relaxation to thermal equilibrium)
  - For a constant drive (plateau), the `nonadiabatic_cgme` method produces constant populations while `redfield` and `secular_lindblad` produce oscillating populations
  - The `secular_lindblad` density matrix stays positive at all times
  - The `nonadiabatic_cgme` density matrix stays positive at all times

**Estimated effort:** 2–3 days.

## Work Package 6: $R_{1\rho}$ Extraction and Comparison

**New file:** `r1rho.py`

This module takes the output of the simulator and extracts the rotating-frame relaxation rate.

**Implementation:**

```
class R1rhoExtractor:
    def __init__(self, result, target_spin_index, method_name):
        """
        result: Result object from Simulator.run()
        target_spin_index: which spin's magnetization to track
        method_name: for labeling
        """
```

**Must compute:**

1. **Transverse magnetization decay** under the spin-lock. Extract $\langle I_x(t) \rangle = \text{Tr}[\rho(t) I_x]$ or equivalently the relevant coherences from the density matrix.

2. **Fit the decay to a single exponential** $M(t) = M_0 e^{-R_{1\rho} t}$ using least-squares fitting over the plateau region (excluding the ramp-up and ramp-down). The fitted decay constant is $R_{1\rho}$.

3. **Repeat for multiple spin-lock powers** $\omega_1$ to build the $R_{1\rho}$ dispersion curve.

4. **Compare against published data.** Load experimental $R_{1\rho}$ values (from a data file or hardcoded from the paper) and plot alongside the three theoretical predictions.

**Deliverables:**
- `r1rho.py` with `R1rhoExtractor` and `R1rhoDispersion` classes
- `test_r1rho.py` verifying: correct exponential fitting on synthetic data, correct extraction of $R_{1\rho}$ from a known analytical case

**Estimated effort:** 1–2 days.

## Testing Strategy

Each work package has its own test file. In addition, there are two integration tests:

**Integration Test 1: Zero-drive equivalence.**
With no spin-lock ($\omega_1 = 0$), all three methods must produce identical dynamics — pure relaxation to thermal equilibrium. The nonadiabatic coefficients should be $b_k = 0$ for all $k > 0$, and $\sigma_{\text{nad}} = |0\rangle\langle 0|$.

**Integration Test 2: Plateau stationarity.**
During the plateau of the spin-lock (constant $V$, $dV/dt = 0$):
- The `nonadiabatic_cgme` populations $|b_k|^2$ must be exactly constant.
- The `redfield` and `secular_lindblad` populations $|c_k|^2$ will oscillate (Rabi oscillations in the Dirac basis).
- This is the single clearest qualitative test that the methods differ.

**Integration Test 3: Known analytical limit.**
For a single spin-1/2 in a spin-lock field with pure $T_1$ relaxation and no dephasing, the analytical $R_{1\rho}$ is known:
$$R_{1\rho} = R_1 \cos^2\theta + R_2 \sin^2\theta$$
All three methods must reproduce this for well-separated levels (where non-secular terms and nonadiabatic corrections are negligible).

## What This Codebase Will Not Do

This simulator operates at the level of a specific few-spin system with a specific bath model. It does not implement the general nonadiabatic Nakajima-Zwanzig framework, which requires the full memory kernel and is BQP-hard for large systems. It implements the Born-Markov-coarse-grained reduction of that framework — the nonadiabatic CGME — which is the classically tractable approximation relevant for the NMR experiment.

It also does not handle near-degenerate levels (Gap 4 from the technical gaps document), non-Markovian baths, or non-perturbative driving. These are limitations of the underlying theory, not of the code.

The codebase is a tool for Phase 0 (feasibility estimate) and Phase 3 (numerical comparison) of the NMR $R_{1\rho}$ experiment plan. It answers the question: for a specific NMR spin system with known parameters, do the standard and nonadiabatic frameworks predict measurably different $R_{1\rho}$ dispersion curves?
